## Honest Feedback

**Friction Point 1: ปัญหา No.6 ซ้อนกันเกินไป**
ปัญหา No.6 มี 4-5 ชั้น: Intent → Invite → Quota → Model → Token แต่ละอันแก้แล้วเจออีก ทำให้ session ยืดมาก ถ้าเราทำ pre-flight checklist ก่อนสร้าง Discord bot จะช่วยได้มาก (Intent ✓ API Key valid ✓ Bot invited ✓)

**Friction Point 2: Token security ยังอ่อน**
.env.gemini เก็บ token แบบ plaintext ไม่มีการ rotate หรือ audit เลย ควรมี secret scanning หรืออย่างน้อย warn เมื่อ token อยู่ใน public-facing script

**Friction Point 3: Bot restart loop ทำให้ Discord throttle**
restart bot บ่อยเกิน (debug loop) จนโดน Discord rate limit ควร test locally ก่อน ไม่ restart บน production bot ขณะ debug

## Lessons Learned

1. **Gemini API quota เต็มง่ายมาก** — free tier สำหรับ gemini-2.5-pro หมดเร็วมาก ใช้ CLI แทนสำหรับ production
2. **Discord bot ใหม่ต้อง verify 3 อย่างก่อน**: Intent enabled → API key valid → Bot invited to right server
3. **Token ที่หลุดใน chat ต้อง revoke ทันที** ไม่ว่าจะดูเหมือนไม่สำคัญแค่ไหน
4. **RAW event ≠ messageCreate** — ถ้า RAW ขึ้นแต่ messageCreate ไม่ตอบ ให้ดูว่า event ทั้งหมดเป็น bot message หรือเปล่า
