## Key Code Changes & Architecture Decisions
1. **FTS5 Sanitization**: SQLite's FTS5 MATCH queries crash on special characters. We refactored query parsing to escape double/single quotes and special characters, transforming `replace(/[\"']/g, " ")` into a comprehensive sanitizer.
2. **GPU Bypass for AMD Vega 56**: BGE-M3 reindexing on Maclab failed with `Caused GPU Timeout Error` (command buffer status 5). We configured CPU-only mode (`-ngl 0 --parallel 1` using Accelerate framework), achieving **22ms per request (400x speedup)**.
3. **Decoupled Relational and Local Vector Storage**: We implemented a hybrid search engine utilizing lexical SQLite FTS5 for query match and FNV-1a hashing for local similarity projections, keeping runtime lightweight.

## Honest Feedback
1. **Subagent Sandbox Limits**: Subagents lack execution permissions for `.git` log/diff searches, leading to time-consuming manual text parsing of `activity.log`.
2. **Google Family Quota limits**: Rotating family accounts is moot due to pooled quota limits, forcing a shift to Codex dual-lane setup.
3. **Stale Handles on LanceDB**: Long-running servers keep stale table handles after reindexing, requiring service restarts to reload vector counts.

---

## Lessons Learned
- **Bypass Faulty GPUs**: For BERT embeddings (`bge-m3`) on Apple Intel Macs with AMD GPUs, use CPU-only configuration (`-ngl 0` via Accelerate framework) to prevent driver hangs.
- **Sanitize FTS5 Operators**: SQLite FTS5 MATCH queries must strip special characters (e.g., `:` and `-`) to prevent runtime query crashes.
- **Workspace Folder Isolation**: In shared classmate repositories, isolate files under subdirectories to avoid branch merge namespace collisions.
