## Key Takeaways (Technical Architecture)
วิดีโอนี้อธิบายถึงกระบวนการ "ปลุกชีพ" (Awakening) ของ Oracle AI Agent และการใช้ Gemini ในกระบวนการทำงานร่วมกัน:

1. **The `/awaken` Protocol**:
   - เป็นคำสั่งเริ่มต้น (Bootstrapping) สำหรับติดตั้งระบบ Oracle, Commands, และ Agents เข้าสู่ Repository ใหม่
   - ทำหน้าที่สร้างโครงสร้างพื้นฐานของความจำ (Memory Layer) ให้พร้อมใช้งาน

2. **The `/trace` Command**:
   - ระบบค้นหาความรู้ (Knowledge Retrieval) จาก Oracle Knowledge Base
   - ช่วยให้ Agent ดึงข้อมูลที่เคยเรียนรู้กลับมาใช้ได้เมื่อต้องการ (On-demand Context)

3. **Gemini`s Role (Deep Research & Transcription)**:
   - Gemini (No.6) ถูกกำหนดบทบาทให้เป็นผู้ทำ Deep Research และถอดเทป (Transcribe) วิดีโอ YouTube
   - แสดงถึงการแบ่งงาน (Delegation) ที่ชัดเจน: ใช้ Model ที่เก่งการค้นหา/วิดีโอ (Gemini) แยกกับ Model ที่เก่งเรื่อง Code/Logic (เช่น Claude หรือ Qwen)

## 🧠 Core Oracle Philosophy
1. **Nothing is Deleted**: บันทึกประวัติศาสตร์ทุกอย่าง ห้ามลบข้อมูลความจำเก่า (Append-only memory)
2. **Patterns Over Intentions**: สังเกตพฤติกรรมและรูปแบบการทำงานจริง มากกว่าแค่เป้าหมายที่ตั้งไว้เบื้องต้น
