**When**: 2026-06-07

## What We Learned

### 1. การรัน Puppeteer บราวเซอร์ด้วยสิทธิ์ Root บน LXC
เมื่อรันคำสั่งเรนเดอร์ PDF จาก Markdown เช่น `npx md-to-pdf` Puppeteer ที่ทำงานข้างใต้จะเปิดใช้ Chromium ซึ่งจะพังทันทีด้วยข้อผิดพลาดเกี่ยวกับความปลอดภัยของ Sandbox เนื่องจากรันบนสิทธิ์ root
- **แนวทางแก้ไข**: จำเป็นต้องระบุอาร์กิวเมนต์ `--launch-options '{"args": ["--no-sandbox"]}'` เพื่อบังคับให้ Puppeteer ข้ามการใช้ sandbox

### 2. การสอบทานขอบเขตการแจ้งเตือน (Alert Dispatch Logic)
เวลาต้องการเปลี่ยนเส้นทางการแจ้งเตือนข่าวสาร (เช่น ย้ายห้องของสแปมแจ้งเตือน) การแก้ไขตัวแปรช่องปลายทางหลักเพียงจุดเดียวอาจไม่พอ หากสคริปต์มีการพ่วงฟังก์ชันส่งสำเนาเข้าช่องทางอื่นๆ เช่น DM ของผู้ดูแล 
- **แนวทางแก้ไข**: ต้องตรวจสอบโค้ดในส่วนของการกระจายข้อความ (dispatch/notify) ทั้งหมด และกำจัดช่องทางที่ไม่ต้องการออกให้ครบถ้วนเพื่อเลี่ยงปัญหาบอทส่งสแปม

### 3. การรักษาความโปร่งใสผ่าน CLI Tmux Pane
การตอบสนองผ่านช่องแชทคู่วิเคราะห์ (Session Chat) ไม่ทำให้เกิดร่องรอยการพิมพ์หรือการโต้ตอบที่มองเห็นได้บนหน้าจอ tmux ทำงานหลัก ทำให้ผู้ดูแล Homelab (บอส Bo) ไม่เห็นว่าเราคุยหรือบรีฟงานกัน
- **แนวทางแก้ไข**: เมื่อมีการประสานงานหรือส่งสคริปต์ให้เพื่อนร่วมงาน ให้รันผ่านคำสั่ง `maw hey <pane> "message"` บนเทอร์มินัลจริงเสมอ เพื่อให้เกิดความโปร่งใสและตรวจสอบความเคลื่อนไหวได้ง่าย
